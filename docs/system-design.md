# IV. 시스템 설계

## 1. 기능 설명

Vori 는 **개인 지출의 합리성을 판정하는 자기관리 웹/모바일 앱**이다. 사용자가 지출 정보(금액·카테고리·시간대) 와 사유(왜 썼는지)를 입력하면, 시스템이 4가지 알고리즘을 동시에 적용해 **합리성 시그널 (🟢 green / ⚪ gray / 🔴 red)** 과 **스탯 가감치** 를 반환한다.

### 핵심 기능

| 분류 | 기능 | 설명 |
|------|------|------|
| **입력** | 지출 등록 | 금액, 카테고리(식비·쇼핑·교통 등), 시간대, 자유 사유 |
| **판정** | 4 방식 합리성 판정 | A·B·C·D 4 알고리즘으로 동시 판정, 결과 비교 |
| **출력** | 시그널 + 스탯 | green +3 / gray +1 / red −2 (사유 보너스 +1 가능) |
| **출력** | 유사 사례 (방식 B) | 임베딩 기반 k-NN 으로 가장 가까운 과거 라벨 사례 3개 |
| **출력** | 적용 룰 (방식 C) | 어떤 규칙이 발동되어 판정되었는지 명시 |
| **히스토리** | 지출 이력 + 통계 | 일·주·월별 시그널 분포, 누적 스탯 추이 |
| **게임화** | 스탯 누적 시각화 | 합리성 점수가 캐릭터·레벨로 시각화 |

### 4 방식 알고리즘

| 방식 | 이름 | 핵심 기술 | 강점 |
|------|------|---------|------|
| A | Gemini 직접 판정 | Google Gemini 2.5 Flash API | 자연어 이해 |
| B | 임베딩 + k-NN | `jhgan/ko-sroberta-multitask` 한국어 임베딩 + 라벨 DB 22건 | 한국어 도메인 특화 |
| C | 규칙 엔진 | 명시적 룰 8개 (시간대·카테고리·금액비·사유) | 결정적·빠름 |
| D | 하이브리드 | C 확정 룰 우선 → 모호 시 B → 보수 선택 | 정확도 + 안정성 |

### 데이터 흐름

```
사용자 입력
  ├─ 지출 (금액, 카테고리, 시간)
  └─ 맥락 (사유, 평균 대비 금액비)
      │
      ▼
  ┌─────────────────────────────────┐
  │  4 방식 동시 실행 (A/B/C/D)       │
  └─────────────────────────────────┘
      │
      ▼
  ┌──────────────────────────┐
  │  결과 통합·비교           │
  │  - 시그널 (green/gray/red) │
  │  - 스탯 가감              │
  │  - 유사 사례 3개          │
  │  - 적용 룰 목록           │
  └──────────────────────────┘
      │
      ▼
  사용자 화면 + DB 저장
```

## 2. 개발 환경, 도구 및 언어

| 구분 | 내역 | 상세내역 |
|------|------|---------|
| **서버** | 운영체제(OS) | 개발: macOS Sequoia 15 / 배포: Linux (Railway) |
| | 플랫폼 | Python 3.12 |
| | 프레임워크 | FastAPI 0.115 (REST API), Uvicorn (ASGI) |
| | 패키지 관리 | pip + venv |
| **클라이언트** | Web Browser | Chrome / Safari / Firefox 최신 |
| | 언어 | TypeScript 5, JavaScript ES2024 |
| | 프레임워크 | React 19 + Next.js 16 (App Router) |
| | 스타일링 | Tailwind CSS v4 + shadcn/ui |
| | 상태 관리 | React Server Components + Zustand (클라이언트 상태) |
| **디바이스** | 데스크탑 | PC / Mac (개발·시연) |
| | 모바일 (Phase 2) | iOS 16+, Android 12+ |
| **DBMS** | DBMS | PostgreSQL 16 (배포) / SQLite (개발) |
| | ORM | SQLAlchemy 2.0 |
| | 관리도구 | Supabase Studio (배포) / DB Browser for SQLite (개발) |
| **AI/ML** | LLM API | Google Gemini API 2.5 Flash (방식 A) |
| | 임베딩 모델 | `jhgan/ko-sroberta-multitask` — 한국어 SBERT (방식 B, 약 400MB) |
| | k-NN 라이브러리 | scikit-learn (cosine similarity) |
| **모바일** | Phase 1 | PWA — `~/Projects/vori/src/web/` Next.js 빌드 결과를 모바일 브라우저에서 "홈 화면에 추가" |
| | Phase 2 | React Native (Expo SDK 50+) — 컴포넌트 70% 재활용 가능 |
| **개발 도구** | IDE | VS Code, Cursor |
| | 버전 관리 | Git + GitHub (private repo) |
| | API 테스트 | FastAPI 자동 docs (Swagger UI), HTTPie |
| | 백엔드 배포 | Railway ($5/월 hobby) |
| | 프론트 배포 | Vercel (Hobby 무료) |
| **인증** | 방식 | NextAuth + Google OAuth |
